# Contributing to TrashFire Public UI

Thank you for reviewing the TrashFire UI! This document explains how to provide feedback and suggestions.

We welcome feedback on:

- **Accessibility** - Keyboard navigation, screen readers
- **Mobile experience** - Responsive design issues

Please don't suggest:
- Adding back API endpoints or backend code

- Security improvements (this is a demo branch)

## ğŸ“ How to Provide Feedback

### Option 1: GitHub Issues

If this repo is public, create an issue with:
- **Title:** `[UI] Brief description`
- **Labels:** `ui-review`, `demo-branch`
- **Description:** Follow the format in `UI_UX_NOTES.md`

### Option 2: Direct Feedback

Send feedback to the maintainer with:
- Screenshots (if applicable)
- Browser/device info
- Steps to reproduce (if it's a bug)
- Your suggestion for improvement

## ğŸ¨ UI Improvement Suggestions

### Format

```markdown
**Component:** [Component Name]
**Current:** [What it looks like now]
**Suggestion:** [What you'd change]
**Rationale:** [Why this would be better]
**Mockup/Screenshot:** [If you have one]
```

### Example

```markdown
**Component:** Dashboard Project Cards
**Current:** Cards are 3 columns on desktop, all same size
**Suggestion:** Make active projects larger, add hover animation
**Rationale:** Helps users quickly identify active work
**Mockup/Screenshot:** [Attach image]
```
